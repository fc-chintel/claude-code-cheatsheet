Claude Code 早見表▊

ターミナルではじめて Claude Code を触る人のための、実用リファレンス。下のタブでジャンルを切り替えられます。

// 用語だけ先に：「セッション」＝Claude Code との1回の会話。「スラッシュコマンド」＝会話中に / から打つ命令。「スキル」＝特定の作業を任せる専用ワークフロー（詳しくは「スキル」タブへ）。コマンドはクリックでコピーできます。
quickstart — 最初の3ステップ
STEP 01
ターミナルを開く
Windows なら「ターミナル」または PowerShell を起動する。
STEP 02
作業フォルダへ移動
cd フォルダのパス と打つ。フォルダを窓にドラッグするとパスが入る。
STEP 03
会話を始める
claude と打つだけ。あとは日本語で話しかけるだけでOK。

👆 クリックしてジャンルを切り替え（← → キーでも移動できます）

00

ターミナルの基本移動

Claude Code の前に、フォルダを行き来する最低限の操作。Windows(PowerShell)・Mac 共通で使えます。

コマンドこれは何をする？
cd パス指定したフォルダへ移動する。cd .. で一つ上の階層へ戻る。
ls今いるフォルダの中身（ファイル・フォルダ）を一覧表示する。
pwd今どこのフォルダにいるかのパスを表示する。「迷子」になったときに。
mkdir 名前新しいフォルダを作る。
01

セッションの開始・再開

Claude Code を立ち上げる／過去の会話に戻る、ターミナルで打つコマンド。まず覚える系。

コマンドこれは何をする？
claude今いるフォルダで、新しい会話を始める。
claude "やりたいこと"最初の指示を渡した状態で、新しい会話をいきなり開始する。
claude --continue-c今いるフォルダの一番最近の会話に、一覧を挟まず即座に戻る。「さっきの続き」に。
claude --resume-r過去の会話の一覧（ピッカー）を出し、選んで開く。どれか選びたいときに。
claude -p "質問"対話モードに入らず、1回だけ質問して答えを表示し終了する。サッと調べたいときに。
claude --no-session-persistenceこの回の会話を履歴に残さない（-p と併用）。一回きり・残したくない作業に。
02

会話中によく使うコマンド

会話の中で / を打つと出るスラッシュコマンド。使用頻度の高い順。

コマンドこれは何をする？
/clear会話をリセットして新しい話題を始める。前の会話は消えず --resume で開ける。
/compact会話が長くなったとき、内容を要約して軽くする（同じ会話のまま続けたいとき）。
/context今どれだけ会話の「記憶容量」を使っているかを可視化する。重くなってきたときに。
/help使い方のヘルプ・コマンド一覧を表示する。
/model使う AI モデルを切り替える。引数なしで実行すると選択メニューが出る。
/usage今のセッションの利用料金・使用量を表示する（/cost でも同じ）。
/permissionsファイル変更やコマンド実行のたびに確認するか、のルールを管理する。
/initこのプロジェクト用の説明ファイル CLAUDE.md を作る。新フォルダで最初にやると良い。SKILL
03

キーボード操作

マウスを使わず、キーだけでサッと操作する。太字は特に出番が多いもの。

キーこれは何をする？
EscClaude が応答・作業している最中に途中で止め、指示を出し直す。安全
Esc×2入力欄が空のとき、以前の状態に戻す「巻き戻しメニュー」を開く。やり直しに便利。
Ctrl+C1回目は入力を消す（実行中なら中断）。空の状態でもう1回押すと終了。
Ctrl+DClaude Code を終了する。
Shift+Tab権限モードを切り替える（計画だけ立てる「プランモード」／編集を自動承認 など）。
!先頭その行をそのままターミナルのコマンドとして実行する（例: !git status）。
@ファイル名の候補が出て、会話中に特定のファイルを指し示せる。
Shift+Enter改行して複数行の指示を入力する（多くのターミナルでそのまま使える）。
↑過去に入力したメッセージを呼び戻す。
Ctrl+L画面表示が崩れたときに、描画をきれいに引き直す（会話内容は消えない）。
04

作業を見直す・やり直す

コードを書いてもらったあと、確認したり、失敗を戻したりするときに。SKILL は「スキル」という仕組みで動く高機能コマンド（一覧は「スキル」タブへ）。

コマンドこれは何をする？
/rewind会話やコードの変更を以前の状態に巻き戻す。「元に戻したい」ときの安全ネット。安全
/diff今どんな変更が加えられたかを、差分表示で確認する。
/code-review今の変更にバグや改善点がないか、Claude にレビューさせる。--fix で指摘を自動修正、--comment でPRにインラインコメント。ultra を付けると複数エージェントによる本格レビューになる（時間・コストがかかる）。SKILLおすすめ
/verify変更が「本当に動くか」を、実際にアプリを起動・操作して確認する。テストが通っていても機能自体が壊れていることはあるので、コミット前の仕上げに。SKILLおすすめ
/runプロジェクトのアプリを実際に起動し、動いている様子を見せてもらう。SKILL
/simplifyバグ探しはせず、コードの重複や無駄を整理してもらう。SKILL
/security-review変更にセキュリティ上の懸念（情報漏洩・認証の穴 など）がないか確認する。SKILL
/doctorClaude Code のインストールや設定に問題がないか診断する。動かないときに。
05

スキル一覧

コードの品質まわりのスキル（/code-review など）は「レビュー」タブにまとめてあります。ここではそれ以外の、知っておくと世界が広がるスキルを紹介します。

スキルとは？ 特定の作業をまとめて任せられる専用ワークフロー。/名前 で明示的に呼び出せるほか、頼み方によっては呼ばなくても Claude が自動で使うことがある（例：グラフを頼むと自動で dataviz の指針が働く）。スラッシュコマンドより手順が凝っていて、複数ステップの判断が挟まるのが特徴。

凡例： おすすめまず知っておきたいもの 自動発動あり頼み方次第で呼ばなくても働く

調べる・自動化する

スキルこれは何をする？
/deep-research1つの問いに対して複数の Web 検索と裏取りを行い、根拠付きのレポートにまとめる。質問が曖昧だと、先に絞り込みの質問をしてくれる。おすすめ
/loop指示や別のスキルを一定間隔で繰り返す（例: /loop 10m デプロイ終わった？）。間隔を省略すると Claude 自身がペースを決める。
/schedule「毎朝9時に」のように、決まった時刻・周期でクラウド上で自動実行されるタスク（ルーティン）を作る。1回だけの予約実行にも使える。

見た目・成果物を作る

スキルこれは何をする？
/datavizグラフ・ダッシュボード・チャートを作る前に、見やすい配色やレイアウトの指針を適用する。「グラフ作って」と頼むだけでも自動的に働くことが多い。自動発動あり
artifact-design共有可能な Web ページ（Artifact）を作る前のデザイン方針。名指しで呼ぶことは少なく、Artifact 作成時に裏側で参照される。自動発動あり

GitHub 連携

スキルこれは何をする？
/reviewGitHub の Pull Request をレビューする。手元でまだPRになっていない差分をレビューしたいときは /code-review を使う。

Claude Code 自体の設定

スキルこれは何をする？
/update-config権限・環境変数・自動実行（フック）など、Claude Code 自体の設定ファイル（settings.json）をまとめて変更する。
/fewer-permission-prompts過去のやり取りから「確認不要」でよい操作を洗い出し、許可リストに追加する。確認ダイアログが多くて煩わしいときに。
/keybindings-helpキーボードショートカットを自分好みにカスタマイズしたいときに。

開発者向けリファレンス

スキルこれは何をする？
claude-apiClaude API / SDK について聞くと、モデルID・料金・キャッシュなど最新の一次情報を調べて答えてくれる。Claude や Anthropic が話題に出ると自動的に働く。自動発動あり
/claude-in-chromeChrome ブラウザを自動操作する（クリック・フォーム入力・スクリーンショット・コンソールログの確認など）。Web 画面での動作確認を頼むときに。
06

MCP — 外部サービスとつなぐ

GitHub・Slack・ブラウザなど、Claude Code の外にあるサービスや道具とつなぐための仕組み。

MCP（Model Context Protocol）とは？ Claude のようなAIと、外部のツール・データソースをつなぐための共通規格。たとえるなら「差し込み口」。GitHubやSlack、Google Driveなど、サービスごとの“差し込みパーツ”（MCPサーバー）をつなぐと、Claude がそのサービスを直接操作できるようになる（例：Slackにメッセージを送る、GitHubのIssueを見る、ブラウザを操作する）。

このチートシートを作る作業でも、実はブラウザを自動操作する claude-in-chrome という MCP サーバーを使って動作確認をしています。

Claude Code（ターミナル）での管理

コマンドこれは何をする？
/mcp今つながっている MCP サーバーの一覧確認・接続・認証などを行う。まずはこれで様子を見る。
claude mcp addターミナルから新しい MCP サーバーを追加する。詳しい書き方は claude mcp add --help で確認できる。
.mcp.jsonプロジェクトのフォルダにこのファイルを置くと、そのプロジェクトを開いた人みんなと接続設定を共有できる（チームでの利用に）。

設定の範囲（スコープ）には「自分のPC・このプロジェクトだけ」「.mcp.json でチーム共有」「自分のPC全体・どのプロジェクトでも」の3段階がある。迷ったら /mcp の案内に従えばOK。

代表的な MCP サーバーの例

サービスつなぐと何ができる？
GitHubIssue や Pull Request の確認・作成など、GitHub上の操作を会話の中から直接行える。
Slackチャンネルやスレッドの検索・既読、メッセージの下書き作成・送信など。
Google WorkspaceGmail の検索、Google カレンダーの予定作成、Google Drive のファイル検索・閲覧など。
ブラウザ（Chrome）実際のブラウザを操作して、クリック・フォーム入力・スクリーンショット・コンソールログの確認ができる。
データベース系Postgres・SQLite など手元のデータベースに直接問い合わせて中身を確認できる。
Notion / Linear / Figmaドキュメント・課題管理・デザインファイルなど、開発でよく使う周辺サービスとの連携。

ブラウザ版・デスクトップ版との関係

場所MCP・連携の設定はどこでする？
Claude Code（ターミナル）上の表の通り、/mcp や .mcp.json などターミナル側で個別に設定する。
claude.ai（ブラウザ）／デスクトップアプリ「設定 → コネクタ」画面から Google Drive・Gmail・Slack などをつなぐ。こちらはアカウント単位の設定で、ブラウザ版とデスクトップアプリで共通。

つながり方は自動では引き継がれない。ブラウザ版やデスクトップアプリで Slack や Google Drive をつないでも、ターミナルの Claude Code には自動で反映されない。それぞれの場所で別々に接続する必要がある（Mac/WSL 環境では、デスクトップアプリの設定を一度だけ Claude Code 側に取り込むコマンドが用意されていることもあるが、常時同期されるわけではない）。

07

発展編

慣れてきたら覚えると便利。最初は読み飛ばしてOK。

コマンドこれは何をする？
/plan大きな変更の前に、計画だけ立てさせてから実行の許可を出す「プランモード」に入る。
/memoryプロジェクトの説明ファイル CLAUDE.md を編集する。
/agents特定の作業を任せる「サブエージェント」を作成・管理する。並行して調べ物をさせたいときなどに。
/code-review ultraクラウド上で複数エージェントが並行してレビューする「ウルトラレビュー」。PR番号 を付けると GitHub の PR も対象にできる。課金・時間がかかるぶん本格的。
claude updateClaude Code 自体を最新バージョンに更新する。
claude --model 名前起動時に使うモデルを指定する（例: sonnet / opus）。

$ うまく頼むコツ

  • 具体的に頼む。「バグ直して」より「ログインを押すとエラーになる、原因を調べて直して」。状況が詳しいほど精度が上がる。
  • 1回に1つの目的。大きすぎる依頼は /plan で計画から。まとめて頼むより、区切って進める方が確実。
  • 材料を渡す。ファイルは @ で指定、エラーやログは ! で実行して共有すると、Claude がそのまま読んで対応できる。
  • 遠慮なく試す。間違えても Esc で止められ、/rewind で戻せる。壊す心配より、まず動かしてみる。
